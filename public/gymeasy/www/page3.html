How to Use

This application is completely done in pure Javascript. Known there as Vanilla Javascript. Some things I had to build:

  • It started with a library of useful functions, similar to JQuery,
    where you can use the $ () syntax to select DOM elements and apply functionality to them.
  • The Carroussel of images
  • The Scroll for divisions with Text
  • The Library to work with Local Storage as a Database

Project Requirements

  • Designing, Defining, and Triggering CSS3 Transitions To assemble the Exercise Carousel I used some Transitions. Especially when making the slide of the images
  • Designing, Defining, and Triggering CSS3 Transformations. the entire page uses transforms for upper case text. And the Cube in the workout page uses 3D transforms.
  • Designing, Defining, and Triggering CSS3 Animations. Especially when making the slide of the images
  • JavaScript, AJAX and LocalStorage.

    The Applicative receives the Exercises from the REST API. 'https://gymeasy.herokuapp.com/exercises' It is an REST API I have created for my Senior Project. It has a database with several exercises with description and other information

    All information in this applicative is stored in the Local Storage. The Exercises, the Workouts (Routines) the association (routines vs Exercises).

    The information remains in storage. So the user will always be able to get the same information from their Workouts, even if they turn off their browser or mobile and load the page the other day.

The only external package I use is RequireJS, because to work with ES6 and import modules from one page to another I had problems. Browsers do not allow this type of configuration because of security. In this case, to be able to share my modules between different pages I use RequireJs (https://requirejs.org/).
